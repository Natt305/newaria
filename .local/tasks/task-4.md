# Add SDXL Image Generation Support

## What & Why

The current image generator uses Cloudflare Flux 1 Schnell, which is optimized for speed but has poor prompt fidelity for complex character details (hairstyle structure, exact color values, etc.). Cloudflare Workers AI also supports SDXL-family models that follow detailed character prompts much more accurately and support a negative_prompt parameter to explicitly suppress unwanted features.

The goal is to make the entire image pipeline work identically for SDXL as it does for Flux — same NSFW retry logic, same regenerate button, same /generate command, same prompt enrichment — just with a configurable model and optional negative prompt support.

## Done looks like

- A new env var `CF_IMAGE_MODEL` selects the generation model (default: current Flux model). Supported values: `flux` (alias for Flux 1 Schnell), `sdxl` (SDXL base 1.0), `sdxl-lightning` (SDXL Lightning), `dreamshaper` (DreamShaper 8 LCM).
- When an SDXL-family model is active, `enhance_image_prompt` returns a `(positive_prompt, negative_prompt)` tuple; negative_prompt is generated by the same LLM call by asking it to output `POSITIVE: ... / NEGATIVE: ...` format. When Flux is active, negative_prompt is None.
- `cloudflare_ai.generate_image(prompt, negative_prompt=None)` accepts the optional negative_prompt and includes it in the request payload for SDXL models. Per-model payload differences (parameter names, resolution, step count, guidance scale) are handled inside `cloudflare_ai.py`.
- Both bot.py call sites (conversational image generation and `/generate` slash command) pass the negative_prompt through.
- The regenerate button in views.py stores and re-uses the negative_prompt from the original generation.
- NSFW retry logic and all error sentinels (API_KEY_ERROR, MODEL_ERROR, NSFW) continue to work identically across all models.
- `launcher.py` logs which image model is active at startup.

## Out of scope

- ControlNet or image-to-image support.
- Switching models mid-conversation (env var set at startup).
- Adding new slash commands for model selection.

## Tasks

1. **`cloudflare_ai.py` — multi-model support** — Read `CF_IMAGE_MODEL` env var to pick the active model and its preset parameters (model path, resolution, step count, guidance scale, parameter name differences). `generate_image(prompt, negative_prompt=None)` builds the correct payload per model. NSFW retry, error handling, and binary/JSON response parsing remain unchanged.

2. **`groq_ai.py` and `ollama_ai.py` — negative prompt generation** — `enhance_image_prompt` detects whether SDXL is active (via a helper that reads `CF_IMAGE_MODEL`). When SDXL is active, the system prompt instructs the LLM to output `POSITIVE: <prompt>` and `NEGATIVE: <negative_prompt>` on separate lines; the function parses this and returns `(positive, negative)`. When Flux is active, returns `(prompt, None)` as before. The negative prompt should cover quality defects (blur, distortion, watermarks) AND character-specific anti-features (wrong hair colors, unwanted hairstyle elements like ahoge if not present in reference).

3. **`ai_backend.py` — proxy update** — Update `enhance_image_prompt` proxy to forward the new return type and all parameters correctly.

4. **`bot.py` — wire negative_prompt through both call sites** — In `process_chat`, unpack `(enriched_prompt, negative_prompt)` from `enhance_image_prompt` and pass `negative_prompt` to `cloudflare_ai.generate_image`. In `/generate` command, same treatment (negative_prompt is None for the direct command path since no LLM enhancement is called there).

5. **`views.py` — regenerate button** — Update `GenerateView` to store `negative_prompt` alongside `prompt` and pass it to `cloudflare_ai.generate_image` on regeneration.

6. **`launcher.py` — log active image model** — Read `CF_IMAGE_MODEL` at startup and log which model is active.

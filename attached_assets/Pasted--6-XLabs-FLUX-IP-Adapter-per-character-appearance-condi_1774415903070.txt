#6 - XLabs FLUX IP-Adapter: per-character appearance conditioning
What & Why
Current multi-character img2img uses a single reference image as the latent seed. At
0.75 strength FLUX is so anchored to that one-character seed that a second character
never appears. IP-Adapter solves this at the conditioning level: instead of encoding
a reference into the latent, it extracts CLIP image features and injects them directly
into the UNet attention layers — giving each character their own high-quality appearance
stream independent of the latent.
For multi-character scenes, XLabsApplyFluxIPAdapter nodes are chained — one per
reference image. Each character's features flow through their own IP-Adapter pass. The
base latent is a plain EmptyLatentImage (txt2img), so there is no img2img seed competing
with or overriding the second character.

Prerequisites (user must install on ComfyUI server)
Custom node: x-flux-comfyui (XLabs-AI/x-flux-comfyui) via ComfyUI Manager orgit clone into custom_nodes + python setup.py
Model weights:ip_adapter.safetensors from XLabs-AI/flux-ip-adapter → ComfyUI/models/xlabs/ipadapters/
clip_l.safetensors from openai/clip-vit-large-patch14 → ComfyUI/models/clip_vision/
Set new tokens in tokens.txt (see below)
New tokens.txt entries
COMFYUI_IPADAPTER=ip_adapter.safetensors
COMFYUI_CLIP_VISION=clip_l.safetensors
# COMFYUI_IPADAPTER_STRENGTH=0.8
# COMFYUI_IPADAPTER_GUIDANCE=3.5

COMFYUI_IPADAPTER and COMFYUI_CLIP_VISION gate the feature — if either is unset
or empty, the existing img2img/txt2img path is used unchanged (no regression).

Workflow — _build_ipadapter_workflow() in comfyui_ai.py
Node graph (for N reference images, N ≥ 1)
Node "1"  UnetLoaderGGUF         ← COMFYUI_GGUF
Node "2"  CLIPLoader             ← COMFYUI_CLIP, type="flux2"
Node "3"  VAELoader              ← COMFYUI_VAE
Node "4"  CLIPTextEncode         ← positive prompt (node "2")
Node "5"  CLIPTextEncode         ← negative prompt (node "2")
Node "6"  EmptyLatentImage       ← width, height (txt2img base — no img2img latent)
Node "14" CLIPVisionLoader       ← COMFYUI_CLIP_VISION
Node "16" XLabsLoadIPAdapter     ← COMFYUI_IPADAPTER

For each reference image i (0-indexed), add:
  Node f"20{i}" LoadImage        ← uploaded reference image name
  Node f"21{i}" XLabsApplyFluxIPAdapter
        inputs:
          model       ← "1" for i=0, else f"21{i-1}" (chain)
          ipadapter   ← "16"
          clip_vision ← "14"
          image       ← f"20{i}"
          strength    ← COMFYUI_IPADAPTER_STRENGTH (default 0.8)
          true_gs     ← COMFYUI_IPADAPTER_GUIDANCE (default 3.5)

Node "13" ModelSamplingFlux
        inputs:
          model       ← f"21{N-1}" (last IP-Adapter in chain)
          width/height as before
Node "7"  KSampler
        inputs:
          model       ← "13"
          latent_image← "6"
          denoise     ← 1.0 (full txt2img — no img2img)
Node "8"  VAEDecode
Node "9"  SaveImage

Prompt prefix
When using IP-Adapter, change the prompt prefix from"referencing the provided image, " to"character appearance conditioned by reference images, ".

Fallback
If any reference image upload fails, fall back to txt2img (not img2img), since
IP-Adapter is the primary conditioning path and a partial upload would give
asymmetric results.

# ----------------------------------------------------------------
# 語言嚴格度 / Language-strictness controls
# ----------------------------------------------------------------
# 這些開關控制「事後語言/重複偵測 → 自動重試」這一整套防護機制。
# 預設全部開啟，能擋下 Mistral Nemo Celeste 的兩個典型錯誤：
#   (a) `_xing_xing_xing…` token 迴圈
#   (b) 用戶說英文時模型卻用中文（甚至簡體）回覆
# 若你跑的模型本身就沒有這些問題（例如 Qwen 系列），可以分開關掉
# 對應的檢查以節省一次 round-trip。接受 1/0、on/off、true/false、
# yes/no、enable/disable（不分大小寫）。
#
# These switches gate the post-generation language/repetition recovery
# system. Defaults are all ON because they're cheap and only fire when
# something is actually wrong; turn them off per-model when you don't
# need them. Accepts 1/0, on/off, true/false, yes/no, enable/disable
# (case-insensitive).
#
#   LMSTUDIO_LANG_ENFORCE         事後品質防護的「主開關」。雖然名稱中
#                                 有 LANG，但它涵蓋的範圍 = (1)+(2)+(3)，
#                                 也就是會同時停掉 token 迴圈截斷重試、
#                                 英文/中文不符重試、簡體字偵測重試，並
#                                 停止注入 user_lang 偵測的覆蓋指令。
#                                 Master switch for the response-recovery
#                                 subsystem. Despite the LANG_ENFORCE name
#                                 it is broader than language: off disables
#                                 (1)+(2)+(3) below AND stops injecting
#                                 the user_lang directive into the system
#                                 prompt — i.e. it turns off ALL post-
#                                 generation quality recovery in one knob.
#   LMSTUDIO_REPETITION_RETRY     (1) Mistral 風格 token 迴圈截斷+重試。
#                                 Off = 連截斷都不做，原文直送。
#                                 The repetition-loop retry path.
#   LMSTUDIO_REPETITION_SALVAGE_MIN
#                                 預設 50。當截斷後剩下的可用文字少於這
#                                 個字元數時才會觸發第二次重試 / 失敗
#                                 訊息回傳。設大一點（例如 100）會更積
#                                 極重試，設小（例如 20）會更寬容。
#                                 Min usable salvage length (chars).
#                                 Higher = more aggressive retries.
#   LMSTUDIO_LANG_MISMATCH_RETRY  (2) 用戶英文/模型中文時的重試。
#                                 The EN-user / ZH-reply retry path.
#   LMSTUDIO_SIMPLIFIED_CHECK     (3) 偵測簡體字並重試要求繁體。
#                                 The Simplified→Traditional retry path.
#
# LMSTUDIO_LANG_ENFORCE=on
# LMSTUDIO_REPETITION_RETRY=on
# LMSTUDIO_REPETITION_SALVAGE_MIN=50
# LMSTUDIO_LANG_MISMATCH_RETRY=on
# LMSTUDIO_SIMPLIFIED_CHECK=on

# ----------------------------------------------------------------
# 角色扮演格式 / Roleplay reply format
# ----------------------------------------------------------------
# 控制 LM Studio 模型系統提示中注入的敘事深度指引。五個層級，
# 全部向「更沉浸」的方向調整。預設值依模型族群自動切換：
# Mistral Nemo（Celeste 等純文字提示模型）→ rich
# Qwen / hauhaucs → standard（純文字指引不會注入 Qwen，改用
# 針對 <subtext> 長度的「軟提示」；微調訓練仍主導，軟提示僅
# 在邊際影響）。
#
#   terse     完全關閉格式指引（不影響語言保護、反迴圈機制）。
#             適合自己已輸出正確格式、不需要額外提示的模型。
#             Qwen 路徑：也不注入軟提示。
#
#   brief     最輕量。要求 2–3 句敘事（至少一句描述肢體語言
#             或環境氣氛），不需要示範。適合 token 預算緊的情境。
#             Qwen 路徑：軟提示 → <subtext> 1–2 句。
#
#   standard  要求每次回覆 2–3 個完整段落，對白前後均須有敘事；
#             不含示範例句。
#             Qwen 路徑：軟提示 → <subtext> 2–4 句。
#
#   rich      預設（Mistral）。要求 4–5 個沉浸式段落，含感官
#             細節、動作、肢體語言，以及至少一段內心獨白。
#             系統提示包含四段示範（含內心想法）。最適合
#             Celeste 等角色扮演微調。
#             Qwen 路徑：軟提示 → <subtext> 4–6 句。
#
#   cinematic 最豐富。要求 6–10 個文學風格段落：環境氣氛、
#             延伸內心獨白、慢節奏情感肌理。含五段示範。
#             適合需要高度沉浸的長篇 RP。
#             Qwen 路徑：軟提示 → <subtext> 6–10 句。
#
# Operator-tunable narrative richness for the roleplay-format directive
# injected into plain-prose (non-Qwen) LM Studio models. Five levels,
# all shifted toward more immersive output:
#
#   terse     No directive injected (hard opt-out). Qwen: no soft hint.
#   brief     2–3 sentences, at least one for body language or atmosphere.
#             No example. Qwen soft hint: <subtext> 1–2 sentences.
#   standard  2–3 full paragraphs wrapping every dialogue beat; no example.
#             Qwen soft hint: <subtext> 2–4 sentences.
#   rich      4–5 immersive paragraphs: sensory detail, action, body
#     (default) language, plus at least one paragraph of internal thought.
#             Includes a 4-paragraph example. Qwen soft hint: 4–6 sentences.
#   cinematic 6–10 literary paragraphs: extended internal monologue,
#             environmental atmosphere, slow-burn emotional texture.
#             Includes a 5-paragraph example. Qwen soft hint: 6–10 sentences.
#
# LMSTUDIO_NARRATION_TARGET=

# ----------------------------------------------------------------
# 結構化輸出 / Structured-output (JSON Schema / JSON mode) switches
# ----------------------------------------------------------------
# 這三個開關控制四條工具呼叫是否在請求中加上「結構化輸出」欄位，
# 強制後端只回傳合法的 JSON：generate_suggestions（按鈕建議）、
# extract_memories（長期記憶擷取）、enhance_image_prompt（圖像提示
# 改寫）、generate_image_comment（圖像旁白）。預設全部關閉（off），
# 維持原本的純文字提示行為；打開後若後端或模型不支援該欄位（HTTP
# 4xx 或 SDK 例外提到 response_format / json mode），程式會自動在當
# 前進程內把它停掉並重試一次同一個呼叫，所以打開不會把工作流卡死。
# 既有的「救援解析器」(salvage parser) 仍會作為最後一道安全網，從
# 編號清單或項目符號文字裡盡力撈出按鈕；圖像那兩條呼叫即使開啟結
# 構化輸出，若解析失敗也會回退到原本的純文字結果。接受 1/0、on/off、
# true/false、yes/no（不分大小寫）。
#
# These three flags decide whether the four utility callers
# (generate_suggestions, extract_memories, enhance_image_prompt,
# generate_image_comment) attach a structured-output field — LM
# Studio's response_format JSON Schema, Groq's
# response_format={"type":"json_object"}, or Ollama's format=json —
# to the request so the backend only returns JSON. All default to off
# so existing setups behave exactly as before. If the backend or
# model rejects the field (4xx, or an SDK exception mentioning
# response_format / json mode), the corresponding flag is auto-
# disabled for the rest of the process and the request is retried
# once without it, so flipping these on never breaks a running bot.
# The existing salvage parser still recovers buttons from numbered
# or bulleted prose as a no-op safety net; the two image callers
# also fall back to plain prose when their JSON parser doesn't find
# a usable string. Accepts 1/0, on/off, true/false, yes/no
# (case-insensitive).
#
#   LMSTUDIO_USE_JSON_SCHEMA   default off — sends a strict JSON
#                              Schema (object root with `items`)
#                              for suggestion buttons (count strings,
#                              each ≤80 chars) and memory facts
#                              (0–3 strings, each ≤60 chars), and a
#                              {"text": "..."} schema for the image
#                              prompt and image-comment callers.
#   GROQ_USE_JSON_MODE         default off — sends
#                              response_format={"type":"json_object"}
#                              and asks the model for an
#                              {"items":[...]} document for buttons /
#                              memories, or {"text":"..."} for image
#                              prompt / image-comment.
#   OLLAMA_USE_JSON_FORMAT     default off — sends format=json on
#                              /api/chat and asks the model for an
#                              {"items":[...]} document for buttons /
#                              memories, or {"text":"..."} for image
#                              prompt / image-comment.
#
# LMSTUDIO_USE_JSON_SCHEMA=off
# GROQ_USE_JSON_MODE=off
# OLLAMA_USE_JSON_FORMAT=off
